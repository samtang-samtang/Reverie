# Castloop 技术报告（技术实现）

> 方法：分析 https://www.castloop.ai/ 的 HTML/JS bundle、HTTP 响应头、运行时请求域名，结合元象 XVERSE 公开技术资料。
> 证据等级：【实测】= 抓取验证；【公开】= 官方/媒体；【推断】= 基于证据推断。

---

## 1. 总体架构（推断）

```
                 ┌──────────────────────────────────────────┐
   浏览器 SPA ───▶│  Next.js 前端（App Router, React）          │
   (zustand/      │  路由 /、/play/[storyId]、DIY 等            │
    Framer        └───────┬───────────────────────┬────────────┘
    Motion/i18n)          │ 静态资源               │ 业务/AI 接口
                          ▼                       ▼
              assets.castloop.ai          xland.xverse.cn（业务/剧情 API）
              （腾讯云 COS + CDN，          yuanx-llm-asset.xverse.cn（LLM 资源）
               短剧分镜 .webp）                     │
                                                   ▼
                                    元象自研泛娱乐 MoE 大模型 + 多模态
                                    （文本生成 / AI 生图 / 语音 / 视频）
                                                   │
                                          SSE 流式回传 ──▶ 前端逐字渲染
```

## 2. 前端技术栈【实测】

| 项 | 证据 | 结论 |
|---|---|---|
| 框架 | 响应头 `X-Powered-By: Next.js`；全量 `/_next/static/chunks/app/...` | **Next.js（App Router）** |
| 视图层 | `react-dom`、`__next` 标记（21 处） | **React** SPA |
| 路由 | `/_next/static/chunks/app/play/%5BstoryId%5D/page-*.js` | **动态路由 `/play/[storyId]`**，含 `not-found` |
| 渲染模式 | 首页 SSR 外壳 + 客户端水合，播放页运行时拉数据后渲染（`Initializing... 0%` 进度） | 混合渲染（静态壳 + 客户端数据） |
| 状态管理 | bundle 中检出 `zustand` | **Zustand** |
| 动画 | bundle 中检出 `Framer`（Framer Motion） | **Framer Motion** |
| 国际化 | bundle 中检出 `Locale` | i18n（英文为主） |
| 埋点 | 内嵌 `googletagmanager.com/gtm.js` | **Google Tag Manager** |
| 流式 | bundle 中高频 `sse`/`Stream` 标识 | **SSE（Server-Sent Events）流式接收生成内容** |

构建产物特征：标准 Next 代码分割（`webpack-*.js`、`polyfills-*.js`、按路由 `app/*/page-*.js`、共享 `chunks/*-{hash}.js`），首屏 JS ≈ 90KB 量级（同类 demo 实测对照）。

## 3. 基础设施 / 部署【实测】

- **站点交付**：响应头 `Server: Lego Server`、`X-NWS-LOG-UUID`、`X-Cache-Lookup: Cache Miss/Hit` —— 典型**腾讯云 CDN（NWS 边缘节点）**特征。
- **静态资源存储**：`assets.castloop.ai` 响应头 `Server: tencent-cos`、`x-cos-request-id`、`x-cos-storage-class: MAZ_STANDARD` —— **腾讯云对象存储 COS，多可用区（MAZ）标准存储**，CDN 回源缓存（`Cache Hit`）。
- 结论：**整体托管在腾讯云**（与团队腾讯背景一致）。【实测/推断】

## 4. 内容资源管线【实测/推断】

短剧分镜资源路径形如：

```
https://assets.castloop.ai/xnarrator-game/editor-beta/{storyId}/{chapter}/image/release/{r_hash}/cover/feed_poster.webp
```

可拆解出关键信息：

- `xnarrator-game`：内部**叙事游戏引擎/产品代号**（"X Narrator"）。
- `editor-beta`：剧情/素材由**编辑器**产出（PGC 生产线，含剧情树、分镜、封面）。
- `{storyId}/{chapter}`：按**剧目 + 章节**组织。
- `release/{r_hash}`：**内容版本化**（哈希指纹，便于发布/回滚/缓存失效）。
- `.webp`：统一 **WebP** 压缩，优化加载。

推断：内容由"编辑器 + AI 多模态生成"产出短剧分镜（封面/场景图，乃至视频），落 COS、走 CDN 分发；运行时再由大模型按玩家选择生成文本分支。

## 5. AI / 后端服务【实测 + 公开】

| 域名 | 角色 | 证据 |
|---|---|---|
| `xland.xverse.cn` | 业务/剧情后端 API（"XLand"游戏服务） | 【实测：播放页运行时引用】 |
| `yuanx-llm-asset.xverse.cn` | LLM 资源/产物（"yuanx" = 元象） | 【实测】 |

**生成模型（元象自研）**【公开】：
- 泛娱乐领域 **MoE 底座大模型**：以自研 Dense 模型嫁接 MoE 框架做 Post-train，优化推理能力、压缩参数、降低成本。
- 多模态能力：**AI 生图、AI 语音通话、AI 视频**（已在 Saylo 全面落地，Castloop 复用同一能力栈）。
- 编排范式（参照同门"昭阳传"）：**多 Agent 流程编排各角色 + 通用大模型 + 专用"小模型"**，配合**记忆系统**维持长程一致性。Castloop 在此基础上偏"短剧分支叙事"。

**生成交互**【实测/推断】：前端以 **SSE 流式**接收剧情正文（逐字/逐段渲染，降低首字延迟），抉择提交后触发下一段生成；结构化返回章节 / 旁白 / 候选选项以驱动 Story Tree。

## 6. 数据与状态【推断】

- 客户端用 **Zustand** 管理播放状态（当前剧目、章节、历史选择、生成流、钻石余额）。
- 服务端维护**剧情树 / 玩家进度 / 角色记忆 / 账号与钻石**（`xland` 业务库）。
- 内容（剧本设定、分镜素材）以"编辑器产出 → 版本化 → COS/CDN"形式静态化分发，生成内容动态化。

## 7. 与本仓库 demo 的对应关系

本仓库 `castloop-mini` 以**同款前端栈**（Next.js App Router + React + TS）复刻了**互动叙事核心闭环**：

| Castloop 真实 | 本 demo 复刻 |
|---|---|
| Next.js App Router | ✅ 同栈 |
| `xland.xverse.cn` + 元象 MoE 大模型 | `app/api/generate` + OpenAI 兼容接口（实测接入火山方舟 **seed-mini**，可插拔任意 LLM），无 key 走 Mock |
| **SSE 流式生成** | ✅ 已实现：`text/event-stream` 逐字回传旁白（`event:delta`），流末下发选项/章节/画面提示（`event:done`）；并过滤推理模型的 `reasoning_content` |
| 短剧分镜画面（AI 生图，COS/CDN） | ✅ 已实现：每幕由 LLM 额外产出英文画面提示 → **Seedream 文生图**（`/api/image`）实时生成竖屏配图；剧目预生成底图兜底 |
| 短剧**视频**分镜 | 未接（同栈可用 **Seedance** `seedance-*` 文/图生视频，因耗时与成本较高，demo 暂以静态图替代） |
| 全屏影游 UI（图上叠字 + 悬浮控件） | ✅ 已实现：竖屏全屏背景 + 底部文字浮层 + 选项浮层 + 侧边控件（配图开关/显隐/重绘/剧情树） |
| 钻石内购 | 钻石计数 UI |
| Story Tree / 选项 / 自由输入 | ✅ 剧情树抽屉 + 选项 + 自由输入闭环 |

## 8. 技术亮点与可借鉴点

1. **内容静态化 + 生成动态化分离**：精品分镜素材走 CDN（快、省算力），仅文本分支实时生成（个性化）。
2. **版本化内容指纹**（`release/{hash}`）便于灰度、回滚、缓存。
3. **SSE 流式**优化生成体验。
4. **自研 MoE + 多模态 + 多 Agent**构成壁垒（叙事一致性 + 成本控制）。
5. **腾讯云 COS(MAZ)+CDN** 保障海外可用性与高可用。
